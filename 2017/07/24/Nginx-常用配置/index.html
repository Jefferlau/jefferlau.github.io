Nginx 安装

使用 Docker Nginx 镜像

  1. 拉取 Nginx 的镜像 docker pull nginx:stable
  2. 拉取 Nginx 配置文件夹 git clone git@github.com:Jefferlau/nginx-config-docker.git
  3. 切换配置到初始安装状态cd nginx-config-docker && git checkout v1.1 && cd ..
  4. 运行 docker run -p 1080:80 --name nginx -v $PWD/nginx-config-docker:/etc/nginx -d nginx:stable 即可启动 Nginx 容器

使用浏览器访问 http://localhost:1080/ 即可看到 Nginx 的欢迎页面。

Docker 容器下演示 CentOS 安装 Nginx

  1. 拉取 CentOS 镜像 docker pull centos
  2. 拉取 Nginx 配置文件夹 git clone git@github.com:Jefferlau/nginx-config-docker.git
  3. 创建容器并映射端口 docker run -it -p 1080:80 --name nginx -v $PWD/nginx-config-docker:/etc/nginx -d centos /bin/bash
  4. 连接容器命令行docker exec -it nginx /bin/bash
  5. 安装 Nginx

使用浏览器访问 http://localhost:1080/ 即可看到 Nginx 的欢迎页面。

负载均衡

Nginx 提供了以下三种负载均衡机制：

  1. round-robin — （默认）请求以轮询的方式分发到应用服务器，如果后端服务器 down 掉，能自动剔除。
    weight 指定轮询几率，weight 和访问比率成正比，用于后端服务器性能不均的情况。
  2. least-connected — 下一个请求被分配到拥有最少活动连接数的服务器
  3. ip-hash — 使用一个哈希函数，基于客户端 IP 地址判断请求应该被分发到哪个服务器,这样每个访客固定访问一个后端服务器，可以解决 session 的问题。

在之前配置 Java 反向代理的时候 nginx.conf 里 upstream 就是用来定义被 Nginx 代理的服务器组

默认情况下，使用轮询机制，如果设置了权重，将会按照将会按照权重来分配响应的服务。

配置了 least-connected 的负载均衡机制的情况下，Nginx
会把新的请求发送到比较不繁忙的服务器。

ip_hash 机制下客户端 IP 地址被用作 hash key 来判断客户端请求应该发送到哪个服务器，这种方法保证了来自相同客户端的请求总是发送到相同服务器（如果服务器可用的话）

如果有必要把客户端绑定至特定服务器，则可使用ip-hash负载均衡机制。
