管理关系

当使用 JPA 时，实体子生成器可以创建实体之间的关系。

介绍

关系仅在使用 JPA 时有效。 如果您选择使用 Cassandra 或 MongoDB，它们将不可用。

关系在两个实体之间工作，JHipster 将生成以下代码：

  • 在生成的 JPA 实体中管理这种关系
  • 创建正确的 Liquibase 变更日志，以便在数据库中体现关系
  • 生成 AngularJS 前端，以便可以在用户界面中以图形方式管理此关系

JHipster UML 和 JDL Studio

本页介绍如何使用标准命令行界面与 JHipster 建立关系。 如果要创建多个实体和关系，则可能更喜欢使用图形工具。

在这种情况下，有两个选项可用：

  • JHipster UML，它允许您使用 UML 编辑器。
  • 在线工具 JDL Studio，使用 domain-specific 语言创建实体和关系。

您可以通过运行jhipster import-jdl your-jdl-file.jh，使用import-jdl子生成器从 JDL 文件生成具有关系的实体。

可用关系

当我们使用 JPA 时，可以使用通常的一对多，多对一，多对多和一对一的关系：

  1. 双向一对多关系
  2. 单向多对一关系
  3. 单向一对多关系
  4. 两个实体之间两个一对多关系
  5. 多对多关系
  6. 一对一关系
  7. 单向一对一关系

提示： User实体

请注意，由 JHipster 处理的User实体具体可以做：

  • 与该实体的多对一(many-to-one)关系（Car可以与User具有多对一的关系）。 这将在您的新实体存储库中生成特定查询，因此可以实现一个常见的需求，在当前的 security 用户上过滤您的实体。 在生成的 AngularJS 客户端界面上，您将在Car中下拉选择一个User。
  • 与User实体的多对多(many-to-many)和一对一(one-to-one)的关系，但另一个实体必须是关系的所有者（一个 Team 可以与 User 有多对多的关系，但只有团队可以添加/删除用户，用户无法添加/删除团队）。 在 AngularJS 客户端 UI 中，您还可以在多选框中选择User。

多对一的单向关系意味着汽车知道他们的主人，但是反过来车主不知道汽车。

你会做这样的关系有两个原因：

  • 从商业角度来看，您只能以这种方式使用您的实体。 所以你不想有一个让开发人员做一些没有意义的事情 API。
  • 使用 Owner 实体时，考虑到性能问题（因为它不需要管理汽车的集合）。

一对多单向关系意味着 Owner实例可以获取其车辆的集合，但反过来不可以。 这与前面的例子相反。

JHipster 目前默认没有提供这种关系，请参阅#1569了解更多信息。

  • 执行双向映射，并使用它，而无需修改：这是我们推荐的方法，因为它更简单
