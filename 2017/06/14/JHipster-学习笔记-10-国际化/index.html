国际化

介绍

在创建一个新项目期间，您将被问及是否要启用国际化支持。

如果启用它，您将需要选择应用程序的母语。 之后，您可以选择要安装的其他语言。 如果您不想从开始支持任何其他语言，您可以随时通过运行语言子生成器添加语言。

如果您确定不会将此应用程序翻译成另一种语言，则不用启用国际化。

如何添加不支持的新语言？

所有语言都保存在文件夹src/main/webapp/i18n（客户端）和src/main/resources/i18n（服务器端）

以下是安装新语言 new_lang 的步骤：

  1. 将src/main/webapp/i18/en文件夹复制到src/main/webapp/i18/new_lang（这是存储所有前端翻译的地方）
  2. 翻译文件夹src/main/webapp/i18/new_lang下的所有文件
  3. 对于AngularJS 1，将语言代码 new_lang 添加到在 src/main/webapp/app/components/language/language.constants.js 中定义的 LANGUAGES 常量

对于Angular 2+，将语言代码 new_lang 添加到在src/main/webapp/app/shared/language/language.constants.ts中定义的 LANGUAGES 常量

  1. 在src/main/resources/i18n文件夹中，将 messages_en.properties 文件复制到 messages_new_lang.properties（这是服务器端转换的存储位置）
  2. 翻译 messages_new_lang.properties 文件中所有键的值
  3. 对于 AngularJS 1，请在 src/main/webapp/app/components/language/language.filter.js 文件中的过滤器函数 filter('findLanguageFromKey') 中添加新语言的名称。 对于Angular 2+，在 src/main/webapp/app/shared/language/language.pipe.ts 中的 FindLanguageFromKeyPipe 的 languages 变量中添加新语言的名称

新语言 new_lang 现在可以在语言菜单中使用，并且它可以同时在前端 Angular 应用程序和后端 Spring 应用程序中使用。

如何删除现有语言？

以下是删除名为 old_lang 的语言的步骤：

  1. 删除语言文件夹 src/main/webapp/i18/old_lang
  2. 删除常量条目 src/main/webapp/app/components/language/language.constants.js 或 src/main/webapp/app/shared/language/language.constants.ts
  3. 删除 src/main/resources/i18n/messages_old_lang.properties 文件
