JHipster 学习笔记(二) - 在开发中使用 JHipster

发表于 2017-06-12 | 分类于 JHipster | | 阅读次数

来自官网

推荐在开始前先看看官方视频 video tutorial!

通用配置

IDE 配置

如果还没配置 IDE，请移步配置 IDE。

应用配置

默认情况下，JHipster 使用“development” profile，因此可以不改变任何配置。

如果你想获取更多可以用的 profiles 信息，请移步 “Profiles”。

如果想配置特定的 JHipster 属性，请参阅 应用程序属性文件。

运行 Java 服务

使用 “main” Java class 运行

在 IDE 中运行主类 main 方法。

使用 Maven 运行

使用项目包装的 Maven ./mvnw (on Mac OS X/Linux) of mvnw (on Windows)
使用本地已安装的 Maven mvn，更多的 Maven 使用信息请移步http://maven.apache.org/
将会执行默认 Maven 任务, spring-boot:run

使用 Gradle 运行

使用项目包装的 Gradle ./gradlew (on Mac OS X/Linux) of gradlew (on Windows)
使用本地已安装的 Gradle gradle，更多的 Gradle 使用信息请移步 https://gradle.org/
将会执行默认 Gradle 任务, bootRun

使用 Angular

运行 Webpack

只需要运行yarn start或npm start

这样运行有非常美妙的功能：

  • 一旦修改了 HTML / CSS / TypeScript 文件，浏览器就会自动刷新
  • 当您在几个不同的浏览器或设备上测试应用程序时，所有点击、滚动和输入都会在所有屏幕上自动同步

这将会启动:

  • Webpack 自动将 TypeScript 代码编译成 JavaScript 的任务
  • Webpack 模块热加载服务运行在 http://localhost:9060/ (同时代理访问 http://127.0.0.1:8080/api Java 后端)
  • BrowserSync 运行在 http://localhost:9000/，它有一个代理 http://localhost:9060/(Webpack 模块热加载服务)，并且它将同步用户的点击、滚动和输入。
  • BrowserSync UI，它运行在 http://localhost:3001/

运行 Yarn

项目直接依赖关系被配置到package.json中，但是传递依赖关系被定义在yarn.lock文件，它在运行yarn install时生成。

建议将yarn.lock放到源代码版本控制中，以便项目组的所有团队成员的所有依赖关系都具有相同的版本。
再次运行yarn install将重新生成具有最新版本的传递依赖关系的yarn.lock。

其他 Yarn / NPM 任务

无论您使用 Yarn 还是 NPM，都可以使用 yarn 命令作为示例，但可以用 npm 替换它们。

  • yarn lint: 检查 TypeScript 代码风格问题
  • yarn lint:fix: 尝试自动纠正 TypeScript lint 风格问题
  • yarn tsc: 编译 TypeScript 代码
  • yarn test: 使用 Karma 运行单元测试
  • yarn test:watch: 保持 karma 单元测试运行, 代码更改时实时反馈
  • yarn e2e: 使用 Protractor 运行“端到端”测试（仅在生成项目时选择了 Protractor 选项时才有效）

使用数据库

运行数据库

使用像 MySQL, MariaDB, PostgreSQL, MSSQL, MongoDB 或 Cassandra 等非嵌入式数据库时，需要自己安装和配置。

JHipster 推荐的最简单的方法是使用 Docker Compose。 参考 Docker 撰写指南。

如果您希望手动安装和配置数据库，请不要忘记在src/main/resources/config/application-*.yml文件（例如数据库URL，登录名和密码）中配置 Spring Boot 属性。

在开发中使用 H2 数据库

如果选择 H2 数据库，则应用程序中将运行内存中的数据库，默认情况下，您可以通过地址http://localhost:8080/h2-console访问控制台。

要连接到数据库，请选择预配置的选项：

  • Driver Class: org.h2.Driver
  • JDBC URL: jdbc:h2:mem:jhipster
  • User name:
  • Password:

在开发中使用 MySQL, MariaDB 或 PostgreSQL

此选项比使用 H2 更复杂，但有一些重要的好处：

  • 数据在应用程序重新启动时保持不变
  • 应用程序启动的更快一点
  • 可以使用强大的 ./mvnw liquibase：diff目标（见下文）

数据库更新

如果增加或修改了 JPA entity，将需要更新数据库。

JHipster使用Liquibase来管理数据库更新，并将其配置存储在/src/main/resources/config/liquibase/目录中。 使用 Liquibase 有3种方法：使用实体子生成器(entity sub-generator)，使用 liquibase：diff Maven目标，或手动更新配置文件。

使用 Maven liquibase:diff 更新数据库

如果选择在开发中使用 MySQL，MariaDB 或 PostgreSQL，则可以使用 ./mvnw liquibase:diff目标自动生成 changelog。

如果您使用基于磁盘的持久性运行 H2，这个尚不能完美工作，但您可以开始尝试使用它（并向官方发送反馈！）。

Liquibase Hibernate 是一个在 Maven pom.xml 中配置的插件，与 Spring application.yml 文件无关，因此如果更改了默认设置（例如，更改了数据库密码），则需要修改这两个文件。

这是开发工作流程：

  • 修改 JPA entity (增加字段 [field], 关系 [relationship], 等。)
  • 编译应用 (编译 Java 代码)
  • 运行 ./mvnw liquibase:diff (或 ./mvnw compile liquibase:diff)
  • 新的 “change log” 被创建在 src/main/resources/config/liquibase/changelog 文件夹
  • 查看此 “change log” 并将其添加到您的 src/main/resources/config/liquibase/master.xml 文件, 在下次运行应用程序时应用它

如果您使用 Gradle 而不是 Maven，则可以通过运行 ./gradlew liquibaseDiffChangelog来使用相同的工作流程，如果需要，可以更改liquibase.gradle中的数据库配置。

手工修改 change log 更新数据库

如果您喜欢（或需要）手动进行数据库更新，则以下是开发工作流程：

  • 修改 JPA entity (增加字段 [field], 关系 [relationship], 等。)
  • 创建新的 “change log” 在src/main/resources/config/liquibase/changelog 文件夹。该目录中的文件以其创建日期（以 yyyyMMddHHmmss 格式）为前缀，然后具有描述其作用的标题。 例如，20141006152300_added_price_to_product.xml是一个好名字。
  • 添加 “change log” 文件到 src/main/resources/config/liquibase/master.xml 文件, 在下次运行应用程序时应用它

如果想了解更多的 Liquibase 使用信息，请移步http://www.liquibase.org。

国际化

国际化（或i18n）是 JHipster 的一等公民，我们相信应该在项目开始时设置（而不是事后的想法）。

用法真的很简单：

  • 使用 AngularJS 1，感谢 Angular Translate，它为 i18n 提供了一个简单的 AngularJS 指令
  • 使用 AngularJS 2+，感谢 NG2 translate 和一个特定的 JHipster 组件，它与 Angular Translate 相同，并使用相同的文件

如果您需要有关使用语言的更多信息，请阅读官方的安装新语言文档。
