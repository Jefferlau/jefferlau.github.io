使用 Angular

请注意本章节仅供 Angular 2+ 参考，AngularJS 1.x 请移步 使用 AngularJS 文档

工具

Angular 2+ 使用的是 TypeScript 而不是 JavaScript，结果有一些特定的工具是我们有效的工作。 我们的 Angular 2+ 应用的开发工作流程如下，如果您愿意，可以使用 npm 代替 yarn。

  1. 生成应用程序时，将创建文件，并在生成结束时，yarn install任务被触发。
  2. yarn install完成后，将调用 package.json 中的 postInstall 脚本，此步骤将触发 webpack:build 任务。
  3. 现在，您应该根据所选的构建工具（ Maven 或 Gradle ）将所有生成的文件和编译到target文件夹或build文件夹中的 www 文件夹中。
  4. 现在运行./mvnw或./gradlew启动应用程序服务器，它应该运行在localhost:8080这也会为上述步骤编译的客户端代码提供服务。
  5. 现在，在新终端运行yarn start，使用 BrowserSync 启动 Webpack dev-server。 这将负责编译您的 TypeScript 代码，并自动刷新您的浏览器。

如果您开始更改客户端代码而没有yarn start，则不会反映任何内容，因为更改未被编译，因此您需要在更改后手动运行yarn webpack:build:dev或运行yarn start。

您还可以强制 maven / gradle 运行 webpack:build:dev任务，同时通过传递 webpackprofile 配置文件，如./mvnw -Pdev,webpack或./gradlew -Pdev -Pwebpack。 这在运行clean任务后特别有用。

  • 要在浏览器中处理代码，我们建议您使用 Angular Augury，因为您可以轻松地查看路由并调试代码。

ng-jhipster 库

ng-jhipster 库是开源免费的，可以在https://github.com/jhipster/ng-jhipster上找到。

ng-jhipster 库包含 Angular 2+ 应用程序使用的实用程序函数和常用组件。 他们包括：

  • 验证指令
  • 国际化组件
  • 通常使用的管道，如大小写，排序和词语截断
  • Base64，日期和分页处理服务
  • 通知系统（见下文）

通知系统

JHipster 使用自定义通知系统将事件从服务器端发送到客户端，并具有能够在生成的应用程序中使用的具有 i18n 功能的JhiAlertComponent和JhiAlertErrorComponent组件。

默认情况下，当 HTTP 响应发生错误时，JHipster 会显示错误通知。

要显示自定义通知或警报，请在将AlertService注入的控制器，指令或服务后使用以下方法。

使用 Angular CLI

Info: Angular CLI 和 JHipster 可以并行用于开发，并且都有自己的配置文件。 默认情况下，JHipster 在部署应用程序或使用 CI-CD sub-generator 时正在使用自己的配置。

概述

Angular CLI 是开发，搭建和维护 Angular 应用的工具。 JHipster 生成 Angular CLI 配置文件，因此 Angular CLI 工作流可与 JHipster 一起使用。

此集成通过在应用程序根文件夹中生成.angular-cli.json文件并将其依赖项添加到package.json文件中来完成。
